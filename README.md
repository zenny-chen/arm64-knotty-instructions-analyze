# ARMv8 架构在 ARM64 执行模式下的一些疑难指令的解析

ARMv8架构是ARM第一次引入了64位执行模式。它同x86一样，在扩展到64位的同时又能兼容老的32位执行模式，而32位执行模式与上一个ARMv7架构完全兼容。而在ARM64执行模式下则是一套完全崭新的ISA，其中包含了一些与ARMv7、乃至其他处理器架构不太一样的指令形式，这里将会对它们做一些具体描述。

<br />

## 按位逻辑计算的立即数形式
在ARM官方的《ARMv8参考指南》中的C4.1.2（Data Processing -- Immediate）小节中描述了按位逻辑计算的立即数编码形式，在 **`Logical (immediate)`** 这一段中。这种立即数的编码方式与传统指令集的立即数编码方式有所不同，它是由两个 **6比特** 的立即数段构成，分别是 **immr** 和 **imms**。另外在操作数为64位通用目的寄存器的场合下还有一个 **N** 位，同immr和imms一起构成最终的mask立即数。从英文字母表示上我们可以基本猜到，immr表示了循环移位的位数，这里采用的是循环右移；而imms呢，则表示对初始立即数的构成。
下面分别介绍一下immr、imms以及N的含义：

#### imms
指示了当前初始立即数的构成。它由多种模式去构造 **比特1**的生成模式，从而生成目标初始立即数的。而对于操作数是32位寄存器还是64位寄存器，imms所产生的立即数模式则略所不同，因此为了更清晰地表达立即数的构造方式，我们将分别予以介绍。

- 对于操作数为32位通用目的寄存器的情况下，imms由5段有效区间构成，分别是：[0, 30], [32, 46], [48, 54], [56, 58], 60。因此，除了上述这些有效区间，其余的，像31、47、55、59、以及61到63均为无效的imms编码值，如果imms采用这些无效的编码值，处理器在执行该条指令时就会抛出“非法指令”的异常。下面分别介绍这5个区间段所构造出的立即数：

1. [0, 30]：这个区间段所构造出的32位立即数表示有 **(imms + 1)** 个连续的比特1。例如：如果imms为0，那么最终生成的初始立即数为1（即1位比特1）；imms为1，则最终生成的初始立即数为3（即2位比特1）；如果 imms 为 `30`，则最终生成的立即数为 `0x7fff'ffff`（即31个比特1）。

2. [32, 46]：这个区间段所构造出的立即数模式表示为，高16位和低16位各由 **(imms - 31)** 个连续的比特1构成。如果 imms 为 `32`，那么最终生成的立即数为 `0x0001'0001`（即高16位与低16位分别有1个连续比特1）；如果 imms 为 `33`，那么最终生成的立即数为 `0x0003'0003`（即高16位与低16位分别有2个连续比特1）；如果 imms 为 `46`，那么最终生成的立即数为 `0x7fff'7fff`（高16位与低16位分别有15个连续比特1）。

3. [48, 54]：这个区间段所构造出的立即数模式是将一个32位立即数划分为4个区域，每个区域为 **1个字节**。因此，它所构造的立即数为，每个区域字节由 **(imms - 47)** 个连续比特1构成。如果 imms 为 `48`，那么最终生成的立即数为 `0x0101'0101`（即每个区域字节有1个连续比特1）；如果 imms 为 `49`，那么最终生成的立即数为 `0x0303'0303`（即每个区域字节有2个连续比特1）；如果 imms 为 `54`，那么最终生成的立即数为`0x7f7f'7f7f`（即每个区域字节有7个连续比特1）。

4. [56, 58]：这个区间段所构造出的立即数模式是将一个32位立即数划分为8个区域，每个区域为 **半字节 （Nybble）**，即4个比特。因此它所构造的立即数为，每个区域半字节由 **(imms - 55)** 个连续比特1构成。如果 imms 为 `56`，那么最终生成的立即数为0x1111'1111（即每个区域半字节有1个连续比特1）；如果 imms 为 `57`，那么最终生成的立即数为 `0x3333'3333`（即每个区域半字节有2个连续比特1）；如果 imms 为 `58`，那么最终生成的立即数为 `0x7777'7777`（即每个区域半字节有3个连续比特1）。

5. 60：如果 imms 为 `60`，那么所生成的初始立即数为 `0x5555'5555`。

- 对于操作数为64位通用目的寄存器的情况下，则还有两种模式——即 **N** 为 `0` 和 **N** 为1的情况。当 N 为 `0` 时，所产生的立即数的规则与32位寄存器操作数的情况类似，所不同的是该规则将同时作用于一个64位立即数的高32位与低32位；而当N为1时，则直接沿用32位寄存器操作数的[0, 30]段的生成规则，而唯一不同的是，此时其范围被扩展到了 [0, 62]。
因此 **当N为0时**，若 imms 为 `0`，则最终生成的立即数为 `0x0000'0001'0000'0001`；若 imms 为 `30` 时，最终生成的立即数为 `0x7fff'ffff'7fff'ffff`；若imms为32时，最终生成的立即数为0x0001'0001'0001'0001；若imms为60时，最终生成的立即数为 `0x5555'5555'5555'5555`。
**当N为1时**，那整个 imms 则表示有 **(imms + 1)** 个连续的比特 `1` 构成最终的64位立即数。比如，当 imms 为 `0` 时，最终生成的立即数为 `0x0000'0000'0000'0001`（共有1个连续比特1）；当 imms 为 `1` 时，最终生成的立即数为 `0x0000'0000'0000'0003`（共有2个连续比特1）；当 imms 为 `31` 时，最终生成的立即数为 `0x0000'0000'ffff'ffff`（共有32个连续比特1）；当 imms 为 `62` 时，最终生成的立即数为：`0x7fff'ffff'ffff'ffff`（共有63个连续比特1）。
